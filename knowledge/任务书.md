# 毕业设计任务书

## 一、题目信息

**题目名称：** 基于SpringBoot的校园维修单智能管理系统
**研究方向：** 后端开发；任务调度优化算法
**题目来源：** 教师科研课题（生产实践）

## 二、主要任务（150字）

针对校园维修服务管理需求，基于SpringBoot单体多模块架构，设计并实现一套智能化的维修订单管理系统。系统集成微信小程序前端，重点研究并实现多因子权重调度算法（负载50%+速度25%+技能25%），解决传统人工派单效率低、分配不均等问题。通过Spring Security+JWT保障系统安全，利用Redis缓存优化性能，集成阿里云OSS实现文件管理。项目需完成系统开发、算法对比测试与毕业论文撰写，体现技术创新性与工程实践价值。

## 三、具体功能要求（200字）

**用户端功能：**
1. 用户注册登录（支持微信小程序OpenID认证）
2. 在线报修申请（支持图片上传、故障描述）
3. 维修进度实时查询与状态追踪
4. 维修完成后反馈评价功能
5. 个人历史订单管理

**管理端功能：**
6. 管理员登录认证与权限控制
7. 新订单智能分配与接单处理（基于多因子权重算法）
8. 维修状态更新与进度管理
9. 管理员工作量统计与分析
10. 系统运行数据监控

**系统功能：**
11. 基于多因子权重的智能调度算法（负载50%+速度25%+技能25%，确保公平性）
12. 敏感信息脱敏与数据安全保护
13. Redis缓存优化与性能提升

## 六、项目特色与创新点

### 🎯 核心创新点
1. **智能调度算法创新**：从简单负载分配升级为**多因子权重算法**，综合考虑负载（50%）、速度（25%）、技能（25%）三个维度
2. **公平性保障**：50%权重确保基本公平分配，避免"欺负老实人"现象，新管理员有保护机制
3. **技术深度提升**：算法涉及加权模型设计、历史数据分析、系统架构思维，体现系统设计能力
4. **对比实验验证**：新旧算法效果对比，量化体现技术创新价值

### 📊 预期技术成果
- **算法效率优化**：预计平均订单处理效率提升15-25%
- **负载均衡改善**：管理员工作量差异控制在20%以内
- **专业技能匹配**：通过关键词匹配实现专业对口分配，提升维修质量
- **系统性能提升**：Redis缓存优化使查询响应时间缩短30-40%

### 🔬 算法设计亮点
**多因子权重模型：**
```
综合得分 = 负载平衡分×50% + 速度表现分×25% + 技能匹配分×25%
```

**核心设计思想：**
- **负载因素（50%）**：保证基本公平，避免分配不均
- **速度因素（25%）**：适当激励效率，但权重控制在合理范围
- **技能因素（25%）**：实现专业对口，提升维修质量
- **新人保护**：新管理员获得中等评分，确保发展机会

### 🎓 答辩核心优势
- **单一技术深度**：专注做好智能调度算法，体现扎实的技术功底
- **实际应用价值**：解决校园维修真实问题，具有工程实践意义
- **对比实验支撑**：新旧算法效果对比，数据说话，体现科研严谨性
- **最新理论支撑**：引用2023-2024年智能调度算法最新研究成果
